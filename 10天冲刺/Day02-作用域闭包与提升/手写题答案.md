# Day 2 手写题答案

## 题 1：实现 `debounce`

来源：

- 2026 Front End Interview Handbook JavaScript utility functions
- 高频前端手写题

```js
function debounce(fn, wait = 300, immediate = false) {
  let timer = null;

  return function (...args) {
    const context = this;
    const callNow = immediate && timer === null;

    clearTimeout(timer);
    timer = setTimeout(() => {
      timer = null;
      if (!immediate) fn.apply(context, args);
    }, wait);

    if (callNow) fn.apply(context, args);
  };
}
```

说明：

- 闭包保存 `timer`
- `immediate` 控制前沿触发

详细思路：

1. 用闭包保存一个定时器 `timer`
2. 每次触发都先清掉旧定时器
3. 重新开启一个新的定时器
4. 如果是立即执行模式，则在“当前没有定时器”时先执行一次
5. 到点后再重置状态

复杂度：

- 时间复杂度：单次调用 `O(1)`
- 空间复杂度：`O(1)`

易错点：

- `this` 和参数不能丢
- `immediate` 模式下，尾部触发和首部触发的逻辑要分开

## 题 2：实现 `throttle`

来源：

- 2026 Front End Interview Handbook JavaScript utility functions
- 高频前端手写题

```js
function throttle(fn, wait = 300) {
  let lastTime = 0;

  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= wait) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}
```

说明：

- 这是时间戳版
- 面试追问时可以再补定时器版

详细思路：

1. 记录上一次执行时间 `lastTime`
2. 每次触发时取当前时间 `now`
3. 如果 `now - lastTime >= wait`，说明到了可执行窗口
4. 执行函数并更新 `lastTime`

复杂度：

- 时间复杂度：单次调用 `O(1)`
- 空间复杂度：`O(1)`

面试追问点：

- 时间戳版是“先执行”
- 定时器版更像“尾部执行”
- 最佳方案通常是两者结合

## 题 3：实现 `once`

来源：

- 高频前端闭包手写题

```js
function once(fn) {
  let called = false;
  let result;

  return function (...args) {
    if (called) return result;
    called = true;
    result = fn.apply(this, args);
    return result;
  };
}
```

说明：

- 闭包保存 `called`
- 适合讲闭包的实际场景

详细思路：

1. 用闭包保存 `called`
2. 第一次执行时，记录结果
3. 后续不再调用原函数，直接返回缓存结果

复杂度：

- 时间复杂度：单次调用 `O(1)`
- 空间复杂度：`O(1)`

## 题 4：实现 `curry`

来源：

- 2026 Front End Interview Handbook higher-order / curry questions

```js
function curry(fn) {
  function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return function (...restArgs) {
      return curried.apply(this, args.concat(restArgs));
    };
  }

  return curried;
}
```

说明：

- 利用 `fn.length` 判断形参个数
- 面试时可以用它顺带解释高阶函数和闭包

详细思路：

1. 返回一个包装函数 `curried`
2. 每次收集参数后，判断当前参数总数是否够用
3. 如果够了，执行原函数
4. 如果不够，继续返回函数收集剩余参数

复杂度：

- 时间复杂度：和参数收集次数有关
- 空间复杂度：递归闭包带来一定调用层级开销

易错点：

- 不能只支持一次传完
- 不能丢失 `this`
