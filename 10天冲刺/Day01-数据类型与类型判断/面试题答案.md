# Day 1 面试题答案

## 1. JavaScript 数据类型有哪些？`BigInt` 和 `Symbol` 分别解决什么问题？

标准回答：

JavaScript 的数据类型可以分成两大类。

- 原始值类型，也叫基本数据类型：`number`、`string`、`boolean`、`null`、`undefined`、`symbol`、`bigint`
- 对象类型，也叫引用数据类型：`object`

如果展开说，对象类型里又包括几种常见情况：

- 标准普通对象：`Object`
- 标准特殊对象：`Array`、`Date`、`RegExp`、`Math`、`Error` 等
- 非标准特殊对象：`Number`、`String`、`Boolean`
- 可调用/可执行对象：`Function`

其中 `Symbol` 和 `BigInt` 是 ES6 之后面试里很容易被追问的两个点。

`Symbol` 的核心作用是创建唯一值，主要用来避免对象属性名冲突，也可以作为对象的唯一标识键，常见于框架底层协议和元编程场景，比如 `Symbol.iterator`、`Symbol.toStringTag`。

`BigInt` 的核心作用是表示超出 `Number` 安全整数范围的大整数。因为 `Number` 本质上是双精度浮点数，安全整数范围只有 `-(2^53 - 1)` 到 `2^53 - 1`，超过这个范围就可能出现精度丢失，所以像超长 ID、大整数计算这类场景就更适合用 `BigInt`。

面试里还可以顺手补一句：

- `typeof null` 结果是 `object`，这是历史遗留问题，不代表 `null` 是对象
- 数组本质上是对象
- 函数本质上也是对象，只不过它是可调用对象

## 2. `typeof null` 为什么是 `object`？

标准回答：

这是 JavaScript 早期历史遗留问题。`typeof` 底层是根据二进制标记做粗粒度判断，早期实现里对象类型的标记是 `000`，而 `null` 的底层表示恰好也被判断成了这一类，所以 `typeof null` 返回了 `object`。

这不是语义上的正确分类，而是历史兼容结果。现在为了兼容旧代码，规范也没有改。

## 3. `null` 和 `undefined` 的区别是什么？

标准回答：

两者都表示“没有值”，但语义不同。

- `undefined` 更偏向“未定义、未赋值、缺省值”
- `null` 更偏向“开发者主动赋的空值”

常见场景：

- 变量声明了但没赋值，默认是 `undefined`
- 函数没有显式 `return`，返回值是 `undefined`
- 对象上不存在的属性，读取结果通常是 `undefined`
- 需要明确表示“这里现在没有对象、没有数据”时，通常会赋值为 `null`

比较关系上也要分清：

```js
null == undefined // true
null === undefined // false
```

这是因为宽松相等里，`null` 和 `undefined` 被专门规定为相等；但严格相等不做类型转换，所以它们类型不同，结果就是 `false`。

面试里可以再补一句：

- `typeof undefined` 是 `undefined`
- `typeof null` 是 `object`，这是历史遗留问题

## 4. `typeof` 的局限性是什么？为什么检测数组不能只靠 `typeof`？

标准回答：

`typeof` 适合检测原始类型中的大部分值，尤其是：

- `undefined`
- `boolean`
- `number`
- `string`
- `symbol`
- `bigint`
- `function`

它的局限主要有两个：

- `null` 会返回 `object`
- 对绝大多数引用类型都只能返回 `object`，无法细分数组、日期、正则等

所以数组不能只靠 `typeof`，因为：

```js
typeof [] // 'object'
typeof {} // 'object'
```

它无法区分数组和普通对象。

## 5. `instanceof` 的原理是什么？它有哪些局限？

标准回答：

`instanceof` 的原理是沿着实例对象的原型链向上查找，看构造函数的 `prototype` 是否出现在这条链上。

例如：

```js
arr instanceof Array
```

本质是在判断 `Array.prototype` 是否出现在 `arr.__proto__` 这条链上。

局限：

- 不能检测原始类型，比如 `1 instanceof Number` 是 `false`
- 跨 iframe / 跨 realm 场景可能失效，因为不同上下文的构造函数不是同一个引用
- 如果手动改写了原型链，结果也可能变化

## 6. 如何准确判断一个值是不是数组、普通对象、日期对象？

标准回答：

最稳的方案是用：

```js
Object.prototype.toString.call(value)
```

例如：

```js
Object.prototype.toString.call([]) // [object Array]
Object.prototype.toString.call({}) // [object Object]
Object.prototype.toString.call(new Date()) // [object Date]
```

如果专门判断数组，也可以直接用：

```js
Array.isArray(value)
```

这是现代项目里最推荐的数组判断方式。

## 7. `Object.prototype.toString.call` 为什么更稳？

标准回答：

因为它基于对象内部的 `[[Class]]` 标签或内置类型标识，能更细地识别对象的具体种类，而不是像 `typeof` 一样只返回 `object`。

它更稳的原因：

- 可以区分数组、对象、日期、正则、Map、Set 等
- 不受大多数普通原型链关系干扰
- 在跨 iframe 场景下通常比 `instanceof` 更可靠

## 8. `==` 和 `===` 的区别是什么？面试里你推荐哪一个，为什么？

标准回答：

- `===` 是严格相等，不做类型转换
- `==` 是宽松相等，必要时会做隐式类型转换

例如：

```js
1 == '1'  // true
1 === '1' // false
```

实际开发里优先推荐 `===`，因为可读性更好、行为更稳定、坑更少。

但面试里不能只说“都用 `===`”，还要知道 `==` 的规则，因为很多经典题都会考它。

## 9. `[] == ![]` 为什么是 `true`？

标准回答：

分三步看：

1. 先算 `![]`

数组是对象，对象转布尔值永远是 `true`，所以：

```js
![] === false
```

2. 变成：

```js
[] == false
```

3. `==` 比较时，对象和布尔值都会参与转换：

- `false -> 0`
- `[] -> '' -> 0`

最终变成：

```js
0 == 0 // true
```

## 10. `NaN` 为什么不等于自己？如何判断一个值是不是 `NaN`？

标准回答：

`NaN` 表示“不是一个合法数字结果”。按照 IEEE 754 浮点规范，`NaN` 与任何值都不相等，包括它自己。

判断方式推荐：

```js
Number.isNaN(value)
```

不要优先用全局 `isNaN`，因为它会先做类型转换，容易误判：

```js
isNaN('abc') // true
Number.isNaN('abc') // false
```

## 11. `Number()`、`parseInt()`、`parseFloat()` 的区别是什么？

标准回答：

- `Number()`：整体转换，要求整个字符串都能转成合法数字，否则就是 `NaN`
- `parseInt()`：从左到右解析整数，遇到第一个不合法字符就停止
- `parseFloat()`：从左到右解析浮点数，支持一个小数点

例如：

```js
Number('12px') // NaN
parseInt('12px') // 12
parseFloat('12.5px') // 12.5
```

## 12. 为什么 `0.1 + 0.2 !== 0.3`？项目里怎么处理精度问题？

标准回答：

因为 JavaScript 的 `Number` 底层是 IEEE 754 双精度浮点数，很多十进制小数无法被二进制精确表示，像 `0.1` 和 `0.2` 都是近似值，运算后误差累积，所以：

```js
0.1 + 0.2 // 0.30000000000000004
```

项目处理方案：

- 金额类场景统一转成整数再算，比如“元转分”
- 使用高精度库，比如 `decimal.js`、`big.js`
- 展示层再做格式化，不在核心计算里靠 `toFixed` 硬修

## 13. `Object.is()` 和 `===` 的区别是什么？

标准回答：

绝大多数场景下结果一样，但有两个关键差异：

- `NaN`
  - `NaN === NaN` 是 `false`
  - `Object.is(NaN, NaN)` 是 `true`
- `+0` 和 `-0`
  - `+0 === -0` 是 `true`
  - `Object.is(+0, -0)` 是 `false`

所以 `Object.is` 更像“SameValue”比较。

## 14. 超长整型 ID 为什么后端常常返回字符串？前端什么时候该用 `BigInt`？

标准回答：

因为前端普通 `Number` 的安全整数范围有限，如果后端直接返回超长整型数字，前端接收后可能精度丢失。

所以很多后端会把长 ID 作为字符串返回，前端：

- 如果只是展示、传递、拼接，一律按字符串处理最安全
- 如果确实要参与大整数运算，再考虑转成 `BigInt`

注意：`BigInt` 不能和普通 `Number` 直接混算。

## 15. 说几个你见过的隐式转换坑题。

标准回答：

常见的有：

- `[] == ![]`
- `[] + {}`
- `{ } + []`
- `'5' - 2`
- `'5' + 2`
- `null == undefined`
- `[] == 0`
- `'' == 0`

面试答这题的重点不是背题，而是能拆规则。

## 16. 如何判断 `object` 为空？

标准回答：

如果是“普通对象是否没有自有可枚举属性”，常用方法有：

```js
Object.keys(obj).length === 0
```

也可以：

```js
Reflect.ownKeys(obj).length === 0
```

区别是 `Reflect.ownKeys` 会把不可枚举和 `Symbol` 键也算进去，更完整。

## 17. 强制类型转换、隐式类型转换分别有哪些规则？

标准回答：

强制类型转换是开发者主动调用 API：

- `Number()`
- `String()`
- `Boolean()`

隐式转换发生在语言运算过程中，比如：

- `+` 号拼接字符串
- `==` 宽松比较
- `if` 判断中的布尔转换
- 算术运算里的数字转换

典型规则：

- 转布尔：`0`、`''`、`null`、`undefined`、`NaN` 是 `false`
- 对象转原始值：优先 `Symbol.toPrimitive`，再 `valueOf`，再 `toString`
- `+` 遇到字符串倾向拼接
- 其它算术运算更倾向转数字

## 20. JavaScript 变量在内存中的堆栈存储怎么理解？

标准回答：

面试里常见的简化表达是：

- 原始类型值通常直接保存在栈中或按值访问
- 引用类型变量在栈里保存引用地址，真正对象内容在堆里

所以对象赋值本质上复制的是地址，不是对象本身。

注意：这是前端面试中的简化模型，帮助理解“按值传递”和“共享引用”，不用讲成底层 VM 实现细节。

## 21. 如何判断 JavaScript 的数据类型？

标准回答：

一套面试最稳的答法是：

- 原始类型优先用 `typeof`
- 数组优先用 `Array.isArray`
- 精确判断对象种类用 `Object.prototype.toString.call`
- 原型关系判断用 `instanceof`

## 22. ES 每个版本引入了什么内容？

标准回答：

面试不用背全年表，重点讲关键版本：

- ES5：严格模式、`JSON`、数组迭代方法、`Object.defineProperty`
- ES6 / ES2015：`let/const`、箭头函数、模板字符串、解构、Promise、class、module、Map/Set、Symbol
- ES2016：`includes`、幂运算符
- ES2017：`async/await`
- ES2018：对象展开、`Promise.finally`
- ES2019：`flat`、`flatMap`
- ES2020：`BigInt`、可选链、空值合并、`Promise.allSettled`

## 23. `let` 声明变量有哪些特性？

标准回答：

- 块级作用域
- 不会重复声明同名变量
- 有暂时性死区
- 会提升，但提升后不能在声明前访问
- 不会像 `var` 一样挂到全局对象上

## 24. 变量提升和函数提升的优先级怎么理解？

标准回答：

在同一作用域内，函数声明的优先级高于 `var` 声明。

例如：

```js
console.log(a);
var a = 1;
function a() {}
```

创建阶段里：

- 先收集函数声明 `a`
- 再处理 `var a`，但不会覆盖已存在的函数声明

所以第一次输出是函数本身，执行阶段再被赋值为 `1`。

## 25. 如何判断对象相等？

标准回答：

要先定义“相等”的含义：

- 如果比较引用是否相同，直接用 `===`
- 如果比较内容是否一致，要做深比较

常见深比较方案：

- 简单场景：`JSON.stringify`
- 严谨场景：递归比较 key 和 value
- 工程里通常用成熟库，比如 `lodash.isEqual`

## 26. `null` 和 `undefined` 的区别是什么？

标准回答：

- `undefined` 更偏“未定义、缺省、未赋值”
- `null` 更偏“开发者主动表示空值”

例如：

- 变量声明未赋值是 `undefined`
- 函数没有显式返回值，默认返回 `undefined`
- 主动清空对象引用常写成 `null`

宽松比较里：

```js
null == undefined // true
null === undefined // false
```

## 27. 宿主对象、内置对象、原生对象分别是什么？

标准回答：

- 内置对象：ECMAScript 标准自带的对象，比如 `Object`、`Array`、`Date`、`Math`
- 宿主对象：运行环境提供的对象，比如浏览器里的 `window`、`document`，Node 里的 `global`
- 原生对象：很多资料里常和“内置对象”混用，一般指语言规范原生提供的对象

面试里说明“不同资料术语边界略有差异”会更稳。

## 28. 如何区分数组和对象？

标准回答：

推荐：

```js
Array.isArray(value)
```

或者：

```js
Object.prototype.toString.call(value) === '[object Array]'
```

不要只用 `typeof`，因为数组和对象都会返回 `object`。

## 29. 什么是类数组（伪数组），如何转成真实数组？

标准回答：

类数组是“长得像数组，但不是数组”的对象，通常具备：

- `length`
- 按数字索引访问

比如：

- `arguments`
- DOM `NodeList`

转数组常见方法：

```js
Array.from(list)
[...list]
Array.prototype.slice.call(list)
```

## 30. 创建函数的几种方式有哪些？

标准回答：

- 函数声明
- 函数表达式
- 箭头函数
- `new Function()`

其中 `new Function()` 很少在业务里用，因为它会动态解析字符串，性能和安全性都较差。

## 31. 创建对象的几种方式有哪些？

标准回答：

- 对象字面量：`{}`
- `new Object()`
- 构造函数：`new Person()`
- `Object.create(proto)`
- `class` 实例化

一般业务里最常见的是对象字面量、构造函数 / class、`Object.create`。

## 隐式类型转换专项测试答案

这一组建议你按下面顺序来推：

1. 先看有没有字符串参与 `+`
2. 再看对象会不会先转原始值
3. 再看布尔值是不是要转成数字
4. 最后再看 `==` 的比较规则

## 32. 题目：

```js
let result = 100 + true + 21.2 + null + undefined + "Tencent" + [] + null + 9 + false;
console.log(result);
```

答案：

```js
NaNTencentnull9false
```

过程：

- `100 + true` 等于 `101`，因为 `true -> 1`
- `101 + 21.2` 等于 `122.2`
- `122.2 + null` 等于 `122.2`，因为 `null -> 0`
- `122.2 + undefined` 等于 `NaN`，因为 `undefined -> NaN`
- `NaN + "Tencent"` 进入字符串拼接，变成 `"NaNTencent"`
- `[]` 转字符串是 `""`，所以还是 `"NaNTencent"`
- 再拼 `null` 得到 `"NaNTencentnull"`
- 再拼 `9` 得到 `"NaNTencentnull9"`
- 再拼 `false` 得到 `"NaNTencentnull9false"`

核心点：

- `undefined` 一旦参与数字运算，很容易把结果带成 `NaN`
- 只要后面遇到字符串，后续 `+` 基本都会走字符串拼接

## 33. `[] + []` 输出什么？

答案：`""`

过程：

- 两边都是数组对象
- 对象参与 `+` 时，先转原始值
- 空数组转字符串是 `""`
- 最后就是 `"" + ""`

## 34. `[] + {}` 输出什么？

答案：`"[object Object]"`

过程：

- `[] -> ""`
- `{}` 转字符串得到 `"[object Object]"`
- 最终是 `"" + "[object Object]"`

## 35. `{} + []` 输出什么？

答案：在这道题按表达式理解，结果通常是 `"[object Object]"`。

过程：

- `{}` 转字符串是 `"[object Object]"`
- `[]` 转字符串是 `""`
- 最终拼接得到 `"[object Object]"`

提醒：

- 这道题在浏览器控制台单独输入时，可能受语句解析影响
- 面试里最好主动补一句：如果把它当成纯表达式，结果按对象转原始值再拼接来分析

## 36. `[] == false` 输出什么？

答案：`true`

过程：

- `false -> 0`
- `[]` 先转原始值，再转数字
- `[] -> "" -> 0`
- 最终比较的是 `0 == 0`

## 37. `{} == false` 输出什么？

答案：`false`

过程：

- `false -> 0`
- `{}` 先转原始值，通常转成字符串 `"[object Object]"`
- `"[object Object]"` 再转数字是 `NaN`
- 最终是 `NaN == 0`

## 38. `'5' + 3 - 2` 输出什么？

答案：`51`

过程：

- `'5' + 3` 先走字符串拼接，得到 `"53"`
- `"53" - 2` 再走数字运算，得到 `51`

## 39. `'5' - 3 + 2` 输出什么？

答案：`4`

过程：

- `'5' - 3` 时，减法不会拼字符串，会强制转数字
- 变成 `5 - 3 = 2`
- `2 + 2 = 4`

## 40. `null + 1` 输出什么？

答案：`1`

过程：

- `null` 在数字运算里会转成 `0`
- 所以结果是 `0 + 1`

## 41. `undefined + 1` 输出什么？

答案：`NaN`

过程：

- `undefined` 转数字是 `NaN`
- `NaN + 1` 还是 `NaN`

## 42. `null == undefined` 输出什么？

答案：`true`

过程：

- 这是 `==` 里的特殊规则
- `null` 只在宽松比较里和 `undefined` 相等，不和其他值相等

## 43. `null === undefined` 输出什么？

答案：`false`

过程：

- 严格相等不做类型转换
- 一个是 `null` 类型，一个是 `undefined` 类型

## 44. `[] == ![]` 输出什么？

答案：`true`

过程：

- `![]` 先执行
- 所有对象转布尔值都是真，所以 `![]` 是 `false`
- 变成 `[] == false`
- `false -> 0`
- `[] -> "" -> 0`
- 最终 `0 == 0`

## 45. `[1, 2] == '1,2'` 输出什么？

答案：`true`

过程：

- 数组先转原始值
- `[1, 2].toString()` 结果是 `"1,2"`
- 最终变成 `"1,2" == "1,2"`

## 46. `[1] + [2, 3]` 输出什么？

答案：`"12,3"`

过程：

- `[1] -> "1"`
- `[2, 3] -> "2,3"`
- 最后是字符串拼接：`"1" + "2,3"`

## 47. `[] + 1` 输出什么？

答案：`"1"`

过程：

- `[] -> ""`
- 然后 `"" + 1`
- 结果是字符串 `"1"`

## 48. `{} + 1` 输出什么？

答案：按表达式理解通常是 `"[object Object]1"`。

过程：

- `{}` 转字符串得到 `"[object Object]"`
- 再和 `1` 拼接

提醒：

- 和第 34 题一样，这题在不同控制台输入方式下可能出现语句解析差异
- 面试时说清“作为表达式时的分析过程”最重要

## 49. `true + true` 输出什么？

答案：`2`

过程：

- `true -> 1`
- `true -> 1`
- 所以是 `1 + 1`

## 50. `true + 'true'` 输出什么？

答案：`"truetrue"`

过程：

- 左边 `true` 先转字符串 `"true"`
- 最终是字符串拼接

## 51. `1 < 2 < 3` 输出什么？

答案：`true`

过程：

- 比较是从左到右算的
- `1 < 2` 先得到 `true`
- `true < 3` 时，`true -> 1`
- 变成 `1 < 3`

## 52. `3 > 2 > 1` 输出什么？

答案：`false`

过程：

- `3 > 2` 先得到 `true`
- `true > 1` 时，`true -> 1`
- 变成 `1 > 1`

## 53. `[] == 0` 输出什么？

答案：`true`

过程：

- `[] -> "" -> 0`
- 最终是 `0 == 0`

## 54. `'' == 0` 输出什么？

答案：`true`

过程：

- 空字符串转数字是 `0`
- 所以最终是 `0 == 0`

## 55. `'' == []` 输出什么？

答案：`true`

过程：

- `[]` 先转原始值
- `[] -> ""`
- 最终比较 `"" == ""`

## 56. `[] === []` 输出什么？

答案：`false`

过程：

- 严格相等比较对象时，比的是引用地址
- 两个字面量数组不是同一个对象

## 57. `NaN == NaN, NaN === NaN, Object.is(NaN, NaN)` 输出什么？

答案：

```js
false false true
```

过程：

- `NaN` 和任何值都不相等，包括自己
- `Object.is` 对 `NaN` 采用更精确的 SameValue 比较规则

## 58. `+[]` 输出什么？

答案：`0`

过程：

- 一元 `+` 会强制转数字
- `[] -> "" -> 0`

## 59. `+{}` 输出什么？

答案：`NaN`

过程：

- 一元 `+` 会强制转数字
- `{}` 先转原始值，得到 `"[object Object]"`
- 再转数字得到 `NaN`

## 60. `[] ? 'A' : 'B'` 输出什么？

答案：`"A"`

过程：

- 条件判断看真假值
- 所有对象都是真值
- 数组虽然“转数字可能是 0”，但在布尔判断里依然是真

## 61. `'2' > '10'` 输出什么？

答案：`true`

过程：

- 两边都是字符串时，按字典序比较，不是按数值大小比较
- 先比第一位字符：`'2'` 大于 `'1'`

面试总结：

- `+` 既可能是加法，也可能是字符串拼接
- `-`、`*`、`/` 基本都会强制转数字
- 对象参与运算时，通常先走 `toPrimitive`
- `==` 的坑很多，但不是无规律，关键是把每一步转换说出来
