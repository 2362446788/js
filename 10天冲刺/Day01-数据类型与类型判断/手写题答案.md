# Day 1 手写题答案

## 题 1：实现 `getType(value)`

来源：

- 2026 Front End Interview Handbook JavaScript quiz/type questions
- 飞书《大前端面试宝典》Q76/Q79/Q104

```js
function getType(value) {
  if (value === null) return "null";
  const type = typeof value;
  if (type !== "object") return type;
  return Object.prototype.toString.call(value).slice(8, -1).toLowerCase();
}
```

说明：

- `null` 要单独判断
- 基础类型优先走 `typeof`
- 对象细分交给 `Object.prototype.toString.call`

详细思路：

1. `typeof null` 是历史遗留坑，所以必须先单独拦截
2. 原始类型里，除了 `null` 之外，`typeof` 已经足够准确
3. 一旦进入 `object`，就不能再依赖 `typeof`，需要用 `toString.call`
4. `toString.call` 返回形如 `[object Array]` 的字符串，截取中间部分并转成小写即可

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

面试追问点：

- 为什么不用 `instanceof` 做全部判断：
  - 原始类型不适合
  - 跨 iframe 场景不稳定

## 题 2：实现 `isEmptyObject(obj)`

来源：

- 飞书《大前端面试宝典》Q73

```js
function isEmptyObject(obj) {
  if (Object.prototype.toString.call(obj) !== "[object Object]") return false;
  return Reflect.ownKeys(obj).length === 0;
}
```

说明：

- 如果只用 `Object.keys`，`Symbol` 键不会被统计
- 面试时可以顺带说清“空对象”的定义

详细思路：

1. 先确认它是不是普通对象，避免把数组、日期对象也当作“空对象”判断
2. 再统计对象自己的 key 数量
3. 这里用 `Reflect.ownKeys` 是为了同时覆盖：
  - 字符串 key
  - `Symbol` key
  - 不可枚举自有属性

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`，这里的 `n` 指 key 数量

易错点：

- `Object.keys({ [Symbol()]: 1 })` 会返回空数组

## 题 3：实现 `arrayLikeToArray(value)`

来源：

- 飞书《大前端面试宝典》Q105

```js
function arrayLikeToArray(value) {
  if (value == null) return [];
  const length = value.length;
  if (typeof length !== "number" || length < 0) return [];
  return Array.from(value);
}
```

说明：

- 真正高频答法是 `Array.from`
- 如果面试官追问兼容写法，可以补 `[].slice.call(value)`

详细思路：

1. 类数组的核心特征是“像数组但不是数组”，通常具备数字索引和 `length`
2. 先对空值做兜底处理
3. 再校验 `length` 是否像一个合法长度
4. 最后交给 `Array.from`

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

面试追问点：

- `Array.from` 和展开运算符的差异：
  - 展开运算符更偏向可迭代对象
  - `Array.from` 对类数组支持更直接

## 题 4：实现 `preciseAdd(a, b)` / `preciseSub(a, b)`

来源：

- 高频前端手写题
- 结合 Day01 数字精度专题

```js
function preciseAdd(a, b) {
  const aDecimals = (String(a).split(".")[1] || "").length;
  const bDecimals = (String(b).split(".")[1] || "").length;
  const base = 10 ** Math.max(aDecimals, bDecimals);
  return (Math.round(a * base) + Math.round(b * base)) / base;
}

function preciseSub(a, b) {
  const aDecimals = (String(a).split(".")[1] || "").length;
  const bDecimals = (String(b).split(".")[1] || "").length;
  const base = 10 ** Math.max(aDecimals, bDecimals);
  return (Math.round(a * base) - Math.round(b * base)) / base;
}
```

说明：

- 核心思想是统一放大为整数再运算
- 金额场景更推荐“元转分”或用高精度库

详细思路：

1. 先统计两个数字小数部分的位数
2. 取最大位数，构造统一倍数 `base`
3. 把两个小数同时放大成整数
4. 用整数完成加减，再缩小回来

为什么这样有效：

- 浮点误差主要来自二进制无法精确表示某些十进制小数
- 放大成整数后，可以尽量绕开直接小数运算

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

面试追问点：

- 真正金额系统里更稳的做法仍然是：
  - 全程用最小货币单位
  - 或使用高精度计算库
