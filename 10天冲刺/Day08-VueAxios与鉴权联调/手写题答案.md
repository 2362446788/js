## 题 1：实现 `createHttpClient`

来源：

- 2026 前端高频工程化手写题
- 结合 Axios 拦截器专题

```js
function createHttpClient() {
  const requestInterceptors = [];
  const responseInterceptors = [];

  async function request(config) {
    let finalConfig = { ...config };

    for (const interceptor of requestInterceptors) {
      finalConfig = await interceptor(finalConfig);
    }

    let response = await fetch(finalConfig.url, finalConfig);

    for (const interceptor of responseInterceptors) {
      response = await interceptor(response);
    }

    return response;
  }

  request.interceptors = {
    request: {
      use(fn) {
        requestInterceptors.push(fn);
      },
    },
    response: {
      use(fn) {
        responseInterceptors.push(fn);
      },
    },
  };

  return request;
}
```

详细思路：

1. 用两个数组保存请求拦截器和响应拦截器
2. 发请求前，按顺序执行请求拦截器
3. 请求完成后，按顺序执行响应拦截器
4. 最终返回处理后的响应

为什么这题重要：

- 它几乎就是你平时说“Axios 二次封装”的简化版本质

## 题 2：实现 `serializeParams`

来源：

- 高频前端工程化手写题
- 结合 Day08 请求封装

```js
function serializeParams(params) {
  return Object.entries(params)
    .filter(([, value]) => value !== undefined && value !== null)
    .map(([key, value]) => {
      return `${encodeURIComponent(key)}=${encodeURIComponent(value)}`;
    })
    .join("&");
}
```

详细思路：

1. 遍历对象的键值对
2. 过滤空值
3. 用 `encodeURIComponent` 处理特殊字符
4. 拼成 `a=1&b=2` 这种形式

## 题 3：实现 `attachToken`

来源：

- Day08 Token 鉴权专题

```js
function attachToken(config, getToken) {
  const token = getToken();
  return {
    ...config,
    headers: {
      ...(config.headers || {}),
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
    },
  };
}
```

详细思路：

1. 把 token 获取逻辑和请求改造逻辑分开
2. 如果 token 存在，就把它注入请求头
3. 不直接改原对象，而是返回新对象，更稳

## 题 4：实现 `createRequestDeduper`

来源：

- 2026 前端高频工程化手写题

```js
function createRequestDeduper() {
  const pending = new Map();

  return function dedupe(key, requestFn) {
    if (pending.has(key)) {
      return pending.get(key);
    }

    const promise = Promise.resolve()
      .then(requestFn)
      .finally(() => {
        pending.delete(key);
      });

    pending.set(key, promise);
    return promise;
  };
}
```

说明：

- 这个题很适合补“工程判断”
- 重点是相同 key 的并发请求只发一次

详细思路：

1. 用 `Map` 保存正在进行中的请求
2. 如果同一个 key 已存在，直接返回旧 Promise
3. 如果不存在，就执行新的请求函数
4. 请求结束后从 `Map` 中删除，避免永久占用

适用场景：

- 搜索建议
- 用户信息请求
- 页面初始化的公共请求
