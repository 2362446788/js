## 宝典补充题

### 题 1：`async/await + Promise + setTimeout` 输出题

参考图：
- [字节面试题.png](/Users/wangtao/Desktop/resume/JS-珠峰/knowledge/字节面试题.png)

请说出下面代码输出顺序，并解释每一步进栈、出栈和入队：

```js
async function async1() {
  console.log('async1 start');
  await async2();
  console.log('async1 end');
}
async function async2() {
  console.log('async2');
}
console.log('script start');
setTimeout(function () {
  console.log('setTimeout');
}, 0);
async1();
new Promise(function (resolve) {
  console.log('promise1');
  resolve();
}).then(function () {
  console.log('promise2');
});
console.log('script end');
```

### 这题重点考什么

- `async` 函数同步执行到 `await` 为止
- `await` 后续逻辑进入微任务队列
- `Promise` 构造函数立即执行
- `then` 回调进入微任务
- `setTimeout` 回调进入宏任务

### 题 2：DOM 事件监听 + Promise 微任务顺序题

参考图：
- [案例面试题.png](/Users/wangtao/Desktop/resume/JS-珠峰/knowledge/案例面试题.png)

```js
let body = document.body;
body.addEventListener('click', function () {
  Promise.resolve().then(() => {
    console.log(1);
  });
  console.log(2);
});
body.addEventListener('click', function () {
  Promise.resolve().then(() => {
    console.log(3);
  });
  console.log(4);
});
```

点击 body 后输出什么？为什么不是 `2 1 4 3` 以外的其它顺序？

### 这题重点考什么

- 一个点击事件回调本身属于任务
- 同一次点击中，监听器会按注册顺序同步执行
- 每个监听器里创建的 `then` 回调进入微任务队列
- 当前任务结束后再统一清空微任务

- Event Loop 不要只背“宏任务微任务”，要讲“调用栈、任务队列、微任务清空时机”。
- Promise 手写题里，`resolvePromise` 是高频追问点。
- 回答 `async/await` 时，最好顺带提错误处理和串并行控制。
- 输出题解释时，按这套顺序最稳：
  - 哪些代码先同步执行
  - 哪些地方创建了微任务
  - 哪些地方创建了宏任务
  - 当前宏任务结束后先清空哪个队列
