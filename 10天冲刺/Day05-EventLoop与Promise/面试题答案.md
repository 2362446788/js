# Day 5 面试题答案

## 0. 什么是 promise，为什么会有 promise，promise 解决了什么问题？
promise 是 es6 以后新增的一个内置类，用于规范js中的异步编程，也能解决回调地狱的问题；
在 promise 出现之前，js中的异步代码主要是靠回调函数来实现，一个函数执行依赖另一个函数的结果，因此不得不把其放置在函数内部去继续执行新函数，导致了回调地狱；
还能够解决了错误处理分散；可以使用catch在最后进行一次统一的处理，而不是每一次函数执行都需要去处理自己的错误逻辑，将错误处理统一进行管理；

## 1. 什么是同步，什么是异步？

标准回答：

- 同步：当前任务不完成，后面的任务不能开始
- 异步：当前任务发起后，不阻塞后续同步代码，结果在未来某个时刻处理

## 2. 什么是 Event Loop？

标准回答：

Event Loop 是 JavaScript 运行时调度同步任务、微任务、宏任务的一套机制。因为 JS 主线程是单线程，Event Loop 负责在调用栈空闲后，把任务队列中的任务依次取出来执行。微任务的优先级高于宏任务。

## 3. 宏任务和微任务分别有哪些？

标准回答：

常见宏任务：

- `script`
- `setTimeout`
- `setInterval`
- DOM 事件回调
- `postMessage`
- `XMLHttpRequest/Fetch`
- node 中的 `setImmediate`

常见微任务：

- `Promise.then/catch/finally`
- `queueMicrotask`
- `MutationObserver`
- `IntersectionObserver`
- `async/await`
- node 中的 `process.nextTick()`

## 4. 一轮事件循环里，宏任务和微任务的执行顺序是什么？

标准回答：

执行顺序是：

1. 执行一个宏任务
2. 宏任务执行完后，立即清空当前产生的所有微任务
3. 再进入下一轮宏任务

一句话：每轮都是“一个宏任务 + 清空微任务”。

## 5. `Promise.then` 回调为什么是异步的？

标准回答：

因为规范要求 `then` 回调不能同步执行，否则会打乱当前代码的执行一致性，链式调用和错误传播也会变得不可预测。所以 `then` 回调会被放进微任务队列，等当前同步代码完成后再执行。

## 6. Promise 有哪些状态？状态为什么不可逆？

标准回答：

三种状态：

- `pending`
- `fulfilled`
- `rejected`

一旦从 `pending` 变成成功或失败，就不能再变回去，也不能从成功变失败。这样设计是为了保证异步结果稳定，避免多次修改导致回调语义混乱。

## 7. `then` 为什么可以链式调用？

标准回答：

因为 `then` 每次都会返回一个新的 Promise。这个新 Promise 的状态取决于 `then` 回调的执行结果：

- 返回普通值：新 Promise 成功，值就是这个返回值
- 返回 Promise：新 Promise 跟随它
- 抛异常：新 Promise 失败

## 8. 什么是值穿透、异常穿透？

标准回答：

- 值穿透：`then` 里如果不传成功回调，值会继续往后传
- 异常穿透：链条中抛出的错误会一直向后传，直到被 `catch` 捕获

## 9. `Promise.resolve`、`Promise.reject`、`Promise.all`、`Promise.race` 有什么区别？

标准回答：

- `Promise.resolve(x)`：返回一个成功 Promise
- `Promise.reject(e)`：返回一个失败 Promise
- `Promise.all`：全部成功才成功，一个失败就失败
- `Promise.race`：谁先有结果就跟谁

## 10. `async/await` 本质上是什么？

标准回答：

`async/await` 本质上是 Promise 的语法糖。`async` 函数一定返回 Promise，`await` 会暂停当前函数后续逻辑，把后面的代码放到微任务里继续执行。

## 11. `await` 后面如果不是 Promise 会怎样？

标准回答：

会被当成：

```js
Promise.resolve(value)
```

处理。也就是说依然会发生“让出执行权”，后续代码异步进入微任务。

## 12. 浏览器和 Node 的事件循环有什么差异？

标准回答：

浏览器面试里重点掌握“宏任务 + 微任务”即可。Node 更复杂，有多个阶段，比如 timers、poll、check 等，而且还有 `process.nextTick` 这种优先级更高的队列。

## 13. 你怎么定位复杂异步输出题？

标准回答：

我会按固定步骤拆：

1. 找同步代码
2. 找宏任务
3. 找微任务
4. 当前宏任务结束后清空微任务
5. 再执行下一个宏任务

## 14. JS 单线程设计的目的是什么？

标准回答：

JavaScript 设计成单线程，核心是为了避免与 DOM 操作的并发冲突。如果多个线程同时修改 DOM，会导致状态难以协调、复杂度极高。

所以浏览器采用：

- JS 主线程单线程执行
- WebAPI 负责异步能力
- Event Loop 负责调度

## 15. 用 `setTimeout` 实现倒计时，与 `setInterval` 的区别是什么？

标准回答：

`setInterval` 是固定间隔重复触发，但如果回调执行时间过长，可能堆积。用递归 `setTimeout` 可以在当前任务执行完成后再安排下一次，更容易控制误差和停止条件。

## 16. JS 事件循环机制里宏任务和微任务是如何工作的？

标准回答：

可以直接答：

“主线程先执行一个宏任务，比如整段 script。执行过程中可能注册宏任务和微任务。当前宏任务结束后，不会立刻执行下一个宏任务，而是会先把当前所有微任务清空，再进入下一轮宏任务。”

## 17. 事件循环输出题：以下代码输出结果是什么？

标准回答：

这类题没有固定唯一模板答案，但固定拆法同第 13 题。面试官真正考的是你是否知道：

- `Promise` 构造函数同步执行
- `then` 是微任务
- `await` 后续是微任务
- `setTimeout` 是宏任务

## 18 - 22. 事件循环进阶（1）到（5）

标准回答：

这几题的标准练法一致：

1. 标注同步代码

## 24. `async/await + Promise + setTimeout` 输出题答案

输出顺序：

```js
script start
async1 start
async2
promise1
script end
async1 end
promise2
setTimeout
```

解释：

- `script start` 同步
- `async1 start` 同步
- `async2` 同步
- `await async2()` 之后，`async1 end` 进入微任务
- `promise1` 是 Promise 构造函数同步执行
- `then` 里的 `promise2` 进入微任务
- `script end` 同步
- 当前宏任务结束，按入队顺序清空微任务：`async1 end`，再 `promise2`
- 最后执行宏任务 `setTimeout`

## 25. DOM 事件监听 + Promise 微任务顺序题答案

输出顺序：

```js
2
4
1
3
```

解释：

- 一次点击事件触发后，这个点击回调过程是一个任务
- 两个监听器按注册顺序同步执行
- 第一个监听器同步输出 `2`，并把 `1` 放进微任务
- 第二个监听器同步输出 `4`，并把 `3` 放进微任务
- 当前任务结束后，清空微任务，按顺序输出 `1`、`3`
