# Day 5 手写题答案

## 题 1：实现 `Promise.all`

来源：

- 2026 Front End Interview Handbook Promise-related APIs
- 高频前端手写题

```js
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const list = Array.from(promises);
    const result = [];
    let count = 0;

    if (list.length === 0) {
      resolve([]);
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        (value) => {
          result[index] = value;
          count += 1;
          if (count === list.length) resolve(result);
        },
        (reason) => reject(reason)
      );
    });
  });
}
```

详细思路：

1. 先把可迭代对象转成数组，方便拿长度和索引
2. 准备结果数组 `result`
3. 每个任务都用 `Promise.resolve` 包一层，兼容普通值
4. 成功时把值放回对应索引，保证顺序不乱
5. 任意一个失败，立刻整体失败

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 2：实现 `Promise.any`

来源：

- 2026 Front End Interview Handbook Promise-related APIs

```js
function promiseAny(promises) {
  return new Promise((resolve, reject) => {
    const list = Array.from(promises);
    const errors = [];
    let rejectedCount = 0;

    if (list.length === 0) {
      reject(new AggregateError([], "All promises were rejected"));
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        resolve,
        (error) => {
          errors[index] = error;
          rejectedCount += 1;
          if (rejectedCount === list.length) {
            reject(new AggregateError(errors, "All promises were rejected"));
          }
        }
      );
    });
  });
}
```

详细思路：

1. 和 `Promise.all` 相反
2. 只要有一个成功，就直接整体成功
3. 只有当所有任务都失败时，才抛出 `AggregateError`

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 3：实现 `serialRun(tasks)`

来源：

- 高频前端异步控制手写题

```js
function serialRun(tasks) {
  return tasks.reduce((prev, task) => {
    return prev.then((result) => {
      return Promise.resolve(task()).then((value) => result.concat(value));
    });
  }, Promise.resolve([]));
}
```

说明：

- `tasks` 是函数数组，不是 Promise 数组
- 每次等上一个完成后再执行下一个

详细思路：

1. 如果直接传 Promise 数组，它们会在创建时就同时开始
2. 所以真正可串行控制的输入必须是“返回 Promise 的函数”
3. 用 `reduce` 把上一个 Promise 链接到下一个任务上
4. 每一步把结果累积到数组里

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 4：实现 `promiseLimit(tasks, limit)`

来源：

- 2026 前端高频手写题趋势：Promise 并发控制

```js
async function promiseLimit(tasks, limit) {
  const results = [];
  let index = 0;

  async function worker() {
    while (index < tasks.length) {
      const currentIndex = index;
      index += 1;
      results[currentIndex] = await tasks[currentIndex]();
    }
  }

  const workers = Array.from(
    { length: Math.min(limit, tasks.length) },
    () => worker()
  );

  await Promise.all(workers);
  return results;
}
```

说明：

- 这是现在很常见的高级前端手写题
- 面试时一定要说清“任务数组是函数数组，否则无法真正控制开始时机”

详细思路：

1. 用一个共享索引 `index` 表示下一个待执行任务
2. 启动 `limit` 个 worker
3. 每个 worker 在 while 循环中不断领取任务
4. 每完成一个任务，就继续拿下一个
5. 所有 worker 都结束后，说明整体完成

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n + limit)`

面试追问点：

- 如何在其中一个任务失败时中断全部流程
- 如何保留错误信息
