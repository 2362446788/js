# Day 5 手写题答案

## 题 1：实现 `Promise.all`

来源：

- 2026 Front End Interview Handbook Promise-related APIs
- 高频前端手写题

```js
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const list = Array.from(promises);
    const result = [];
    let count = 0;

    if (list.length === 0) {
      resolve([]);
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        (value) => {
          result[index] = value;
          count += 1;
          if (count === list.length) resolve(result);
        },
        (reason) => reject(reason)
      );
    });
  });
}
```

详细思路：

1. 先把可迭代对象转成数组，方便拿长度和索引
2. 准备结果数组 `result`
3. 每个任务都用 `Promise.resolve` 包一层，兼容普通值
4. 成功时把值放回对应索引，保证顺序不乱
5. 任意一个失败，立刻整体失败

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 2：实现 `Promise.any`

来源：

- 2026 Front End Interview Handbook Promise-related APIs

```js
function promiseAny(promises) {
  return new Promise((resolve, reject) => {
    const list = Array.from(promises);
    const errors = [];
    let rejectedCount = 0;

    if (list.length === 0) {
      reject(new AggregateError([], "All promises were rejected"));
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        resolve,
        (error) => {
          errors[index] = error;
          rejectedCount += 1;
          if (rejectedCount === list.length) {
            reject(new AggregateError(errors, "All promises were rejected"));
          }
        }
      );
    });
  });
}
```

详细思路：

1. 和 `Promise.all` 相反
2. 只要有一个成功，就直接整体成功
3. 只有当所有任务都失败时，才抛出 `AggregateError`

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 3：实现 `serialRun(tasks)`

来源：

- 高频前端异步控制手写题

```js
function serialRun(tasks) {
  return tasks.reduce((prev, task) => {
    return prev.then((result) => {
      return Promise.resolve(task()).then((value) => result.concat(value));
    });
  }, Promise.resolve([]));
}
```

说明：

- `tasks` 是函数数组，不是 Promise 数组
- 每次等上一个完成后再执行下一个

详细思路：

1. 如果直接传 Promise 数组，它们会在创建时就同时开始
2. 所以真正可串行控制的输入必须是“返回 Promise 的函数”
3. 用 `reduce` 把上一个 Promise 链接到下一个任务上
4. 每一步把结果累积到数组里

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 题 4：实现 `promiseLimit(tasks, limit)`

来源：

- 2026 前端高频手写题趋势：Promise 并发控制

```js
async function promiseLimit(tasks, limit) {
  const results = [];
  let index = 0;

  async function worker() {
    while (index < tasks.length) {
      const currentIndex = index;
      index += 1;
      results[currentIndex] = await tasks[currentIndex]();
    }
  }

  const workers = Array.from(
    { length: Math.min(limit, tasks.length) },
    () => worker()
  );

  await Promise.all(workers);
  return results;
}
```

说明：

- 这是现在很常见的高级前端手写题
- 面试时一定要说清“任务数组是函数数组，否则无法真正控制开始时机”

详细思路：

1. 用一个共享索引 `index` 表示下一个待执行任务
2. 启动 `limit` 个 worker
3. 每个 worker 在 while 循环中不断领取任务
4. 每完成一个任务，就继续拿下一个
5. 所有 worker 都结束后，说明整体完成

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n + limit)`

面试追问点：

- 如何在其中一个任务失败时中断全部流程
- 如何保留错误信息

## 题 5：实现 `Promise.race`

来源：

- 2026 Front End Interview Handbook Promise-related APIs
- 高频前端手写题

```js
function promiseRace(promises) {
  return new Promise((resolve, reject) => {
    for (const item of promises) {
      Promise.resolve(item).then(resolve, reject);
    }
  });
}
```

详细思路：

1. 接收一个可迭代对象，逐一取出来
2. 每个值都用 `Promise.resolve` 包一层，兼容非 Promise 值
3. 第一个 settled 的结果直接透传给外层 Promise
4. 因为 Promise 的状态一旦确定就不会再变，后续的 resolve/reject 会被忽略

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

面试追问点：

- `Promise.race` 和 `Promise.any` 的区别
  - race：第一个 settled（无论成功或失败）就结束
  - any：第一个 fulfilled 才成功，全部失败才 reject

## 题 6：实现 `Promise.allSettled`

来源：

- 2026 Front End Interview Handbook Promise-related APIs

```js
function promiseAllSettled(promises) {
  return new Promise((resolve) => {
    const list = Array.from(promises);
    const result = [];
    let settledCount = 0;

    if (list.length === 0) {
      resolve([]);
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        (value) => {
          result[index] = { status: 'fulfilled', value };
          settledCount += 1;
          if (settledCount === list.length) resolve(result);
        },
        (reason) => {
          result[index] = { status: 'rejected', reason };
          settledCount += 1;
          if (settledCount === list.length) resolve(result);
        }
      );
    });
  });
}
```

详细思路：

1. 和 `Promise.all` 很像，但永远不会 reject
2. 无论成功还是失败，都记录在结果数组里
3. 成功项格式：`{ status: 'fulfilled', value }`
4. 失败项格式：`{ status: 'rejected', reason }`
5. 用 `Array.from` 转成数组，确保有 `length` 可用
6. 计数到 `list.length` 时整体 resolve

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

面试追问点：

- 和 `Promise.all` 的关键区别：allSettled 永远不会走到 catch，除非你代码本身有语法错误

## 题 7：实现 `promiseAllLimit(promises, limit)`

来源：

- 字节跳动面试题

```js
function promiseAllLimit(promises, limit) {
  return new Promise((resolve, reject) => {
    const list = Array.from(promises);
    const results = [];
    let settledCount = 0;
    let failedCount = 0;

    if (list.length === 0) {
      resolve([]);
      return;
    }

    list.forEach((item, index) => {
      Promise.resolve(item).then(
        (value) => {
          results[index] = value;
          settledCount += 1;
          if (settledCount === list.length) resolve(results);
        },
        () => {
          failedCount += 1;
          if (failedCount >= limit) {
            reject(new Error("There have been " + limit + " failures!"));
            return;
          }
          results[index] = null;
          settledCount += 1;
          if (settledCount === list.length) resolve(results);
        }
      );
    });
  });
}
```

说明：

- 这和题 4 的 `promiseLimit` 完全不同：`allLimit` 控制的是**失败容忍数**，而非并发数
- 所有 Promise 同时发起，不做并发控制

详细思路：

1. 所有 Promise 同时执行，与 `Promise.all` 一致
2. 维护两个计数器：`settledCount`（总 settled 数）和 `failedCount`（失败数）
3. 成功路径：记录值，`settledCount` 累加
4. 失败路径：先检查 `failedCount >= limit`，若达到上限则立刻 reject
5. 未达上限：该位置填 `null`，`settledCount` 累加
6. 当 `settledCount === list.length` 时，所有 Promise 均已 settled，resolve 结果

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

与 `Promise.all` 的对比：

- `Promise.all`：一个失败立刻 reject，零容忍
- `Promise.allLimit(promises, 0)`：等价于 `Promise.all`，零容忍
- `Promise.allLimit(promises, N)`：最多容忍 N 个失败，失败的位填 `null`

面试追问点：

- 如何处理非 Promise 值？
  - 用 `Promise.resolve(item)` 包裹，与 `Promise.all` 一致
- 如果 `limit` 传了非数字或负数？
  - 参考实现中 `limit = +limit; if (isNaN(limit)) limit = 1;`
  - 即 NaN 视为 1，负数按实际值比较（例如 -1 意味失败一次就 reject，因为 `0 >= -1` 为 true）
- 失败位置填 `null` 的语义？
  - 表示「该位置的任务执行过了，但失败了」，和未执行区分开
