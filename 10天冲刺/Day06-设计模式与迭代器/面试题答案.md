# Day 6 面试题答案

## 1. 什么是迭代器协议？

标准回答：

迭代器协议要求对象提供一个 `next()` 方法，每次调用返回一个对象：

```js
{ value: any, done: boolean }
```

`done` 表示是否遍历结束。

## 2. 什么是可迭代对象？哪些内置对象默认可迭代？

标准回答：

可迭代对象是指实现了 `Symbol.iterator` 方法的对象。常见内置可迭代对象：

- Array
- String
- Map
- Set
- TypedArray

## 3. `for...of` 和 `for...in` 的区别是什么？

标准回答：

- `for...of`：遍历可迭代对象的值
- `for...in`：遍历对象的可枚举键名

数组遍历通常优先 `for...of`，对象遍历通常配合 `Object.keys/entries`。

## 4. Generator 的 `yield` 是什么含义？

标准回答：

`yield` 表示“产出一个值并暂停函数执行”，下次调用 `next()` 时从暂停位置继续执行。

## 5. Generator 和 `async/await` 有什么历史关系？

标准回答：

在 `async/await` 普及前，社区常用 `Generator + Promise + 执行器` 来写异步流程控制。`async/await` 可以看成更高级、更易读的异步流程语法糖。

## 6. 什么是发布订阅模式？

标准回答：

发布订阅模式里，发布者和订阅者不直接依赖，中间通过事件中心或消息中心解耦。

## 7. 什么是观察者模式？

标准回答：

观察者模式中，被观察者维护观察者列表，状态变化时直接通知观察者。它们之间是直接依赖关系。

## 8. 发布订阅和观察者模式的区别是什么？

标准回答：

核心区别是有没有中间层：

- 观察者模式：观察者和目标对象直接关联
- 发布订阅模式：通过事件中心解耦

## 9. 你在前端项目里哪里用过事件中心或总线？

标准回答模板：

“我在跨组件通信、全局弹窗控制、登录态广播、刷新列表通知这类场景里用过事件总线。它的好处是减少组件之间的直接依赖，但如果滥用也会让依赖关系变隐式，所以现在更倾向在明确场景下用状态管理或更可追踪的方案。”

## 10. 设计模式在前端里有哪些常见体现？

标准回答：

- 单例：全局弹窗管理器、全局缓存实例
- 工厂：组件创建、请求实例创建
- 观察者 / 发布订阅：事件总线、响应式通知
- 代理：请求代理、图片懒加载代理思路
- 策略：表单校验规则切换

## 11. 单例模式在前端里的例子有哪些？

标准回答：

- 全局 store
- 全局消息提示实例
- 全局埋点上报器
- 全局 WebSocket 连接管理器

## 12. 为什么说设计模式是解决代码组织问题，而不是炫技？

标准回答：

因为设计模式本质上是经验总结，目的是：

- 降低耦合
- 提高复用
- 让结构更清晰

如果只是为了“套模式”，反而会让代码复杂化。
