# Day 6 手写题答案

## 题 1：实现 `EventEmitter`

来源：

- 2026 Front End Interview Handbook Event Emitter
- 高频前端手写题

```js
class EventEmitter {
  constructor() {
    this.events = new Map();
  }

  on(eventName, handler) {
    const list = this.events.get(eventName) || [];
    list.push(handler);
    this.events.set(eventName, list);
  }

  off(eventName, handler) {
    const list = this.events.get(eventName) || [];
    this.events.set(
      eventName,
      list.filter((item) => item !== handler)
    );
  }

  emit(eventName, ...args) {
    const list = this.events.get(eventName) || [];
    list.slice().forEach((handler) => handler(...args));
  }

  once(eventName, handler) {
    const wrapper = (...args) => {
      handler(...args);
      this.off(eventName, wrapper);
    };
    this.on(eventName, wrapper);
  }
}
```

详细思路：

1. 用 `Map` 维护 `事件名 -> 处理函数数组`
2. `on` 负责注册
3. `off` 负责按引用删除
4. `emit` 负责执行
5. `once` 本质是包装一个只执行一次的函数

易错点：

- `emit` 时最好对数组做浅拷贝，避免执行中途修改原数组

## 题 2：实现 `Subject / Observer`

来源：

- 飞书《大前端面试宝典》设计模式相关
- Day06 观察者模式专题

```js
class Subject {
  constructor() {
    this.observers = new Set();
  }

  subscribe(observer) {
    this.observers.add(observer);
  }

  unsubscribe(observer) {
    this.observers.delete(observer);
  }

  notify(payload) {
    this.observers.forEach((observer) => observer.update(payload));
  }
}
```

详细思路：

1. Subject 维护一个观察者集合
2. 订阅时加入，取消订阅时移除
3. `notify` 时遍历所有观察者并调用其 `update`

和发布订阅差异：

- 这里没有中间事件总线
- 观察者直接依赖 Subject

## 题 3：实现 `makeRangeIterator(start, end)`

来源：

- 2026 Front End Interview Handbook JavaScript fundamentals
- Day06 迭代器专题

```js
function makeRangeIterator(start, end) {
  let current = start;

  return {
    next() {
      if (current < end) {
        return { value: current++, done: false };
      }
      return { value: undefined, done: true };
    },
  };
}
```

详细思路：

1. 闭包保存当前游标 `current`
2. 每次 `next()` 都根据当前值返回结果
3. 到边界后返回 `done: true`

## 题 4：给对象增加 `Symbol.iterator`

来源：

- Day06 可迭代对象专题

```js
function attachIterator(obj) {
  obj[Symbol.iterator] = function* () {
    for (const key of Object.keys(this)) {
      yield [key, this[key]];
    }
  };
  return obj;
}
```

说明：

- 这里返回 `[key, value]` 元组
- 面试时也可以改成只遍历值，关键是讲清 `Symbol.iterator`

详细思路：

1. 给对象挂上 `Symbol.iterator`
2. 让它返回一个生成器
3. 生成器内部按照你定义的顺序逐个 `yield`

为什么用生成器方便：

- 代码更短
- 天然符合迭代器协议
