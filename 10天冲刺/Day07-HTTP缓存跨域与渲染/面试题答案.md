# Day 7 面试题答案

## 1. 从输入 URL 到页面渲染完成，发生了什么？

标准回答：

高频标准版可以分 8 步：

1. 输入 URL，浏览器解析协议、域名、端口、路径
2. 查缓存、查 DNS，拿到服务器 IP
3. 建立 TCP 连接，HTTPS 还会做 TLS 握手
4. 浏览器发送 HTTP 请求
5. 服务器处理并返回响应
6. 浏览器解析 HTML，构建 DOM
7. 解析 CSS，构建 CSSOM，再生成渲染树
8. 布局、绘制、合成，页面展示出来

如果 HTML 中有脚本、样式、图片等资源，还会继续发起子资源请求。

## 2. HTTP 和 HTTPS 有什么区别？

标准回答：

- HTTP 明文传输
- HTTPS = HTTP + TLS/SSL，加密、更安全
- HTTPS 能防中间人攻击和内容篡改
- HTTPS 默认端口 443，HTTP 默认端口 80

## 3. 常见 HTTP 状态码有哪些？

标准回答：

- `200` 成功
- `204` 成功但无内容
- `301/302` 重定向
- `304` 协商缓存命中
- `400` 请求有问题
- `401` 未认证
- `403` 无权限
- `404` 资源不存在
- `500` 服务器错误

## 4. 什么是强缓存，什么是协商缓存？

标准回答：

- 强缓存：浏览器直接用本地缓存，不发请求或无需取新资源
- 协商缓存：浏览器带条件请求到服务器，服务器判断资源是否变更，未变更返回 `304`

## 5. `Cache-Control`、`Expires`、`ETag`、`Last-Modified` 的区别是什么？

标准回答：

- `Cache-Control`：HTTP/1.1，强缓存核心，优先级高
- `Expires`：HTTP/1.0，基于绝对时间
- `ETag`：资源唯一标识，更精确的协商缓存
- `Last-Modified`：资源最后修改时间，协商缓存

## 6. 为什么 HTML 页面通常不做强缓存？

标准回答：

因为 HTML 通常是资源入口。如果 HTML 强缓存时间过长，页面即使引用的静态资源已经更新，用户也可能拿不到最新资源引用关系。实际项目里更常见的是：

- HTML 不做长强缓存
- JS/CSS 文件名做 hash，配合强缓存

## 7. 什么是跨域？同源策略限制的到底是什么？

标准回答：

跨域是指协议、域名、端口任一不同。  
同源策略主要限制的是：一个源的脚本不能随意读取另一个源的敏感资源响应。

重点是：浏览器限制的是“读响应”，不一定是“请求发不出去”。

## 8. CORS 是怎么工作的？什么是预检请求？

标准回答：

CORS 是服务器通过响应头显式告诉浏览器“哪些源可以访问我”的机制。

常见头：

- `Access-Control-Allow-Origin`
- `Access-Control-Allow-Methods`
- `Access-Control-Allow-Headers`
- `Access-Control-Allow-Credentials`

预检请求就是浏览器在某些跨域请求前先发一个 `OPTIONS` 请求，确认服务器是否允许真正请求。

## 9. 什么请求会触发 OPTIONS 预检？

标准回答：

当请求不是“简单请求”时，通常会触发预检，比如：

- 使用 `PUT`、`DELETE`
- 自定义请求头
- `Content-Type` 不是简单值

## 10. JSONP 的原理是什么？为什么它有局限？

标准回答：

JSONP 利用 `script` 标签不受同源策略限制，通过动态插入脚本，把服务端返回的数据包装成函数调用来执行。

局限：

- 只支持 `GET`
- 依赖服务端配合
- 安全性差
- 错误处理弱

## 11. 反向代理解决跨域的本质是什么？

标准回答：

本质是“让浏览器以为请求还是发给同源服务器”，再由同源服务器转发到真正后端。因为浏览器只看自己发请求的目标源，不关心服务器内部如何转发。

## 12. 浏览器渲染流程是什么？什么是 CRP？

标准回答：

渲染流程：

- 解析 HTML 得到 DOM
- 解析 CSS 得到 CSSOM
- 合成渲染树
- Layout 布局
- Paint 绘制
- Composite 合成

CRP 指 Critical Rendering Path，中文常说关键渲染路径，就是浏览器把字节变成像素的关键流程。

## 13. 什么是重排和重绘？分别由什么触发？

标准回答：

- 重排（回流）：布局信息改变，比如宽高、位置、节点增删
- 重绘：样式变化但不影响布局，比如颜色、背景

重排一定会引起重绘，重绘不一定引起重排。

## 14. 动画为什么更推荐 `transform` 和 `opacity`？

标准回答：

因为它们通常不会触发布局和大范围绘制，更容易走合成层，性能更好，尤其适合高频动画。

## 15. 你做过哪些前端性能优化？

标准回答模板：

可以按四类讲：

- 网络层：压缩资源、CDN、缓存、懒加载
- 渲染层：减少重排重绘、骨架屏、虚拟列表
- 代码层：按需加载、拆包、减少首屏 JS
- 交互层：节流防抖、请求合并、图片优化

## 16. DOM 节点的 `Attribute` 和 `Property` 有什么区别？

标准回答：

- `Attribute` 是 HTML 标签上的属性，偏静态
- `Property` 是 DOM 对象上的属性，偏运行时状态

例如输入框的 `value`，用户输入后改的是 `property`，不一定同步回标签源码里的 `attribute`。

## 17. DOM 结构操作有哪些常用 API？

标准回答：

- 创建：`createElement`
- 添加：`appendChild`、`append`
- 插入：`insertBefore`
- 移除：`removeChild`、`remove`
- 克隆：`cloneNode`
- 查找：`querySelector`、`getElementById`、`getElementsByClassName`

## 18. DOM 的事件模型是什么？

标准回答：

DOM 事件流分三阶段：

- 捕获阶段
- 目标阶段
- 冒泡阶段

## 19. 事件三要素是什么？

标准回答：

常见答法是：

- 事件源
- 事件类型
- 事件处理函数

## 20. 如何绑定事件和解除事件？

标准回答：

推荐现代写法：

```js
el.addEventListener('click', handler)
el.removeEventListener('click', handler)
```

解绑时必须传同一个函数引用。

## 21. 事件冒泡和事件捕获的区别是什么？如何阻止？

标准回答：

- 捕获：从外到内
- 冒泡：从内到外

阻止冒泡：

```js
event.stopPropagation()
```

阻止默认行为：

```js
event.preventDefault()
```

## 22. 什么是事件委托？

标准回答：

事件委托是把子元素事件统一绑定到父元素上，利用事件冒泡，在父元素中根据事件源判断具体触发目标。

优点：

- 减少事件监听数量
- 对动态新增节点友好

## 23. JavaScript 动画和 CSS3 动画有什么区别？

标准回答：

- CSS 动画更适合简单视觉效果，性能更稳，维护方便
- JS 动画更灵活，适合复杂逻辑控制、拖拽、物理效果

## 24. 如何获取元素位置？

标准回答：

常见：

- `getBoundingClientRect()`
- `offsetTop/offsetLeft`
- 结合 `scrollTop/scrollLeft` 计算页面绝对位置

## 25. `document.write` 和 `innerHTML` 的区别是什么？

标准回答：

- `document.write` 在文档加载完成后调用可能会重写整个页面，不推荐现代项目使用
- `innerHTML` 是修改指定元素的 HTML 内容

## 26. `mouseover` 和 `mouseenter` 的区别是什么？

标准回答：

- `mouseover` 会冒泡，移入子元素时也会触发
- `mouseenter` 不冒泡，更稳定

## 27. 元素拖动实现方案是什么？

标准回答：

通常流程：

1. `mousedown` 记录起始位置
2. `mousemove` 计算位移并更新元素位置
3. `mouseup` 解绑移动事件

## 28. `script` 标签 `async` 和 `defer` 的区别是什么？

标准回答：

- 普通 `script`：下载和执行都会阻塞 HTML 解析
- `defer`：异步下载，HTML 解析完成后按顺序执行
- `async`：异步下载，下载完立即执行，不保证顺序
