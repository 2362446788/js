# Day 4 面试题答案

## 1. JavaScript 常见继承方案有哪些？

标准回答：

常见继承方案有：

- 原型链继承
- 借用构造函数继承
- 寄生组合继承
- ES6 `class extends`

现代项目里通常优先说：

- ES6 `class extends`
- 理论层面知道寄生组合继承最经典

## 2. 原型链继承、借用构造函数、寄生组合继承分别有什么问题？

标准回答：

- 原型链继承
  - 引用类型属性会被实例共享
  - 不能方便给父类传参
- 借用构造函数
  - 能解决共享引用问题
  - 但方法都写在父构造函数里，不能复用
- 寄生组合继承
  - 解决组合继承的重复调用问题
  - ES5 时代最推荐

## 3. ES6 `class extends` 本质上做了什么？

标准回答：

本质还是构造函数 + 原型链继承，只是语法层更像传统类。底层会建立：

- 子类原型对象到父类原型对象的链路
- 子类构造函数到父类构造函数的静态链路

## 4. 浅拷贝和深拷贝的区别是什么？

标准回答：

- 浅拷贝：只复制第一层，内部引用类型仍共享
- 深拷贝：递归复制所有层级，目标和源对象彻底独立

## 5. `Object.assign` 算深拷贝吗？为什么？

标准回答：

不算。它只复制第一层属性值，如果属性值是对象，复制的是引用地址。

## 6. `JSON.parse(JSON.stringify())` 有哪些局限？

标准回答：

- 丢失 `undefined`
- 丢失函数
- 丢失 `Symbol`
- `Date` 会变字符串
- `RegExp` 会变空对象或异常丢失语义
- 无法处理循环引用
- `Map`、`Set` 等特殊结构不适合

## 7. 如何处理循环引用的深拷贝？

标准回答：

常见做法是用 `WeakMap` 记录已经拷贝过的源对象与目标对象映射，递归时如果再次遇到同一个对象，直接返回已缓存结果，避免死循环。

## 8. 如何实现数组扁平化？说几个方案。

标准回答：

常见方案：

- 递归
- `reduce + concat`
- 原生 `flat(Infinity)`
- 栈模拟递归

面试手写时，`reduce + 递归` 最常见。

## 9. 如何实现数组去重？不同数据类型怎么考虑？

标准回答：

最常见：

```js
[...new Set(arr)];
```

如果是对象数组去重，要明确“按什么字段去重”，通常借助 `Map`：

```js
const map = new Map();
arr.filter((item) => !map.has(item.id) && map.set(item.id, true));
```

## 10. 如何把扁平数组转成树结构？

标准回答：

核心思路是先建映射表，再挂载子节点：

1. 先按 `id` 建 `map`
2. 再遍历数组
3. 有 `parentId` 的挂到父节点 `children`
4. 没有 `parentId` 的放根节点数组

## 11. 如何做数据清洗，比如去空值、去重、字段映射？

标准回答：

数据清洗核心是“三步走”：

- 过滤无效项
- 统一字段结构
- 根据业务规则去重或排序

常见工具方法：

- `filter`
- `map`
- `reduce`
- `Set` / `Map`

## 12. 项目里做过哪些复杂数据转换？

标准回答模板：

可以说：

“我做过列表接口字段和页面展示模型不一致的场景。通常我会先做一层适配，把后端字段统一映射成前端视图模型，再做去空值、默认值补齐、状态枚举转换，最后再做分组或树结构处理。这样组件层不直接依赖后端原始结构，维护起来更稳。”

## 13. 以下哪段代码效率更高：数组快速模式 / 字典模式？

标准回答：

结论是：保持数组连续、稳定、同类型，通常更容易被引擎优化为快速模式；如果数组变得稀疏、索引跳跃很大、频繁删改，可能退化成字典模式，性能会下降。

面试里重点不是死背某个引擎实现，而是知道：

- 连续索引更友好
- 稀疏数组和“把数组当对象用”不友好

## 14. 数组去重的方法有哪些？

标准回答：

- `Set`
- `indexOf`
- `includes`
- 双重循环
- `reduce`
- `Map`

项目里优先用 `Set` 或 `Map`，更清晰。

## 15. 清空数组的方法有哪些？

标准回答：

- `arr.length = 0`
- `arr.splice(0, arr.length)`
- `arr = []`

区别：

- 前两种会影响原数组引用
- `arr = []` 只是让变量指向新数组，不会影响其他仍引用旧数组的地方

## 16. JavaScript 数组常见操作方式及方法有哪些？

标准回答：

- 增删：`push`、`pop`、`shift`、`unshift`、`splice`
- 截取/拼接：`slice`、`concat`
- 遍历：`forEach`、`map`、`filter`、`reduce`
- 查找：`find`、`findIndex`、`includes`、`indexOf`
- 排序：`sort`、`reverse`
- 展平：`flat`

## 17. `reduce` 方法怎么用？

标准回答：

`reduce` 是把数组压缩成一个最终结果。签名：

```js
arr.reduce((acc, cur, index, array) => {}, initialValue);
```

常见用途：

- 求和
- 统计频次
- 数组转对象
- 扁平化

## 18. 如何遍历对象？

标准回答：

- `for...in`：遍历可枚举属性，包含继承链上的可枚举属性
- `Object.keys`：自有可枚举键
- `Object.values`
- `Object.entries`
- `Reflect.ownKeys`：更完整

如果面试官问“最常用”，业务里通常是 `Object.keys/entries`。
