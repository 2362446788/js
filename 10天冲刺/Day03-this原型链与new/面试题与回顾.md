## 飞书宝典补充题

14. ES6 的继承和 ES5 的继承有什么区别？

### 这题建议答法

- 先说结论：
  - ES6 `class extends` 只是更清晰的语法糖，本质依然基于原型链。
- 再说区别：
  - 写法更接近传统类
  - `super` 更清晰
  - 必须先 `super()` 再访问 `this`
  - 静态方法继承表达更自然
  - 底层仍然是构造函数 + 原型链 + `Object.setPrototypeOf`

## 宝典补充题

### 题 1：函数的多种角色综合题

参考图：
- [函数多种角色面试题.png](/Users/wangtao/Desktop/resume/JS-珠峰/0422/函数多种角色面试题.png)

请说出下面代码的输出，并逐行解释原因：

```js
function Foo() {
  getName = function () {
    console.log(1);
  };
  return this;
}
Foo.getName = function () {
  console.log(2);
};
Foo.prototype.getName = function () {
  console.log(3);
};
var getName = function () {
  console.log(4);
};
function getName() {
  console.log(5);
}

Foo.getName();
getName();
Foo().getName();
getName();
new Foo.getName();
new Foo().getName();
new new Foo().getName();
```

### 这题重点考什么

- 函数声明和变量声明提升
- 构造函数和普通函数执行差异
- 静态属性方法和原型方法
- `this` 指向
- 运算符优先级，尤其是 `new Foo.getName()` 和 `new Foo().getName()`

### 这题你必须能讲到的点

- `Foo.getName()` 访问的是函数对象身上的静态方法。
- `Foo()` 普通执行时会修改全局 `getName`。
- `new Foo()` 创建实例，实例访问不到私有 `getName`，会沿原型链找 `Foo.prototype.getName`。
- `new Foo.getName()` 等价于 `new (Foo.getName)()`。
- `new Foo().getName()` 等价于 `(new Foo()).getName()`。

## 答题提醒

- 回答原型链时，不要把“继承”和“查找机制”混在一起。
- `this` 一题里，面试官最想看的是你有没有条理。
- 手写 `bind` 时别忘了构造调用场景。
- 遇到综合输出题，不要急着报数字，先拆成：
  - 标识符来自哪
  - 当前是不是 `new`
  - 访问的是静态方法、实例方法还是原型方法
  - 运算符优先级怎么结合
