# Day 3 面试题答案

## 1. `this` 到底怎么判断？请按调用方式分类说。

标准回答：
我们研究的THIS，都是研究函数私有上下文中的THIS;

- 因为全局上下文中的this->window
- 块级上下文中没有自己的this，在此上下文中遇到的this，都是其所处环境(上级上下文)中的this
- ES6中的箭头函数和块级上下文类似，也是没有自己的this，遇到的this也是其上级上下文中的
  THIS是执行主体：通俗来讲，是谁把它执行的，而不是在哪执行的，也不是在哪定义的，所以THIS是谁和在哪执行以及在哪定义都没有直接的关系
  判断 `this` 最稳的方式是按调用方式分：

* 普通函数调用：非严格模式下指向 `window`，严格模式下是 `undefined`
* 对象方法调用：谁调用就指向谁
* 构造函数调用：指向新创建的实例
* `call/apply/bind`：显式指定
* 箭头函数：没有自己的 `this`，继承外层
* DOM 事件回调：通常指向绑定事件的元素

## 2. 普通函数、对象方法、构造函数、箭头函数里的 `this` 有什么区别？

标准回答：

- 普通函数：看是否严格模式
- 对象方法：指向调用它的对象
- 构造函数：指向实例
- 箭头函数：词法绑定，不看调用者

## 3. `call`、`apply`、`bind` 的区别是什么？

标准回答：

- `call`：立即调用，参数逐个传
- `apply`：立即调用，参数数组传
- `bind`：不立即执行，返回一个绑定了 `this` 的新函数

## 4. `bind` 为什么返回一个新函数？

标准回答：

因为它的设计目标不是“立刻执行”，而是“预先绑定 `this` 和部分参数，供以后调用”。所以它更像函数柯里化或函数包装。

## 5. `new` 关键字做了什么？

标准回答：

`new` 大致做 3 件事：

1. 创建一个构造函数的实例对象
2. 用这个新对象作为 `this` 执行构造函数
3. 如果构造函数返回对象，就返回该对象；否则返回新实例

## 6. 为什么箭头函数不能当构造函数？

标准回答：

因为箭头函数没有自己的：

- `this`
- `arguments`
- `prototype`

也不能被 `new` 调用，所以不能作为构造函数。

## 7. 什么是显式原型，什么是隐式原型？

标准回答：

- 显式原型：函数对象上的 `prototype`
- 隐式原型：对象实例上的 `__proto__`

实例的 `__proto__` 通常指向其构造函数的 `prototype`。

## 8. `prototype` 和 `__proto__` 有什么区别？

标准回答：

- `prototype` 是函数作为构造器时给实例共享方法用的原型对象
- `__proto__` 是对象内部的原型引用，用于原型链查找

一句话：

- `prototype` 是“给别人连的”
- `__proto__` 是“自己连到哪的”

## 9. JavaScript 原型链是怎么查找属性的？

标准回答：

访问对象属性时：

1. 先找对象自身属性
2. 没有就沿 `__proto__` 向上找
3. 一直找到 `Object.prototype`
4. 最后到 `null` 仍没找到，就返回 `undefined`

## 11. `Object.create(null)` 创建的对象有什么特殊之处？

标准回答：

它创建的是一个没有原型的纯净对象：

- 没有继承 `Object.prototype`
- 没有 `toString`、`hasOwnProperty` 等方法
- 非常适合做纯字典或哈希表

## 12. 面向对象在 JavaScript 里本质是什么？

标准回答：

JavaScript 的面向对象本质是基于对象和原型链的对象复用机制，不是传统 class-based OO。即使 ES6 有 `class`，底层本质还是构造函数和原型链。

## 13. 原型模式相比把方法写在构造函数里，有什么优缺点？

标准回答：

优点：

- 方法共享，节省内存
- 实例访问统一行为

缺点：

- 公共数据容易被共享误用
- 如果实例需要完全私有的方法或状态，不适合全放原型

## 13.1 多次call、apply、bind会发生什么？

总结：

规律：

```js
一个CALL执行 例如：B.call(A, 40, 30);
// => 让B执行，让B中的this是A，给B传递40/30
多个CALL执行 例如：B.call.call.call(A, 20, 10);
// => 让A执行，让A中的this是20，给A传递10

一个 APPLY 执行 例如：B.apply(A, [40, 30]);
// => 让B执行，让B中的this是A，给B传递数组[40, 30]
多个 APPLY 执行 例如：B.apply.apply.apply(A, [20, 10]);
// => 让A执行，让A中的this是[20, 10]，给A传递空参数（undefined）

一个 BIND 执行 例如：B.bind(A, 40, 30);
// => 不执行B，而是返回一个新函数，让新函数中的this永久锁定为A，并预设参数40/30
多个 BIND 执行 例如：B.bind(A, 40, 30).bind({name:'珠峰'}, 50);
// => 依然返回一个新函数，让新函数中的this依然是A（bind的this一旦绑定不可被修改），并将预设参数累加为40/30/50
```

```js
Function.prototype.call = function call(context, ...params) {
  if (context == null) context = window;
  if (!/^(object|function)$/.test(typeof context)) context = Object(context);
  let self = this,
    key = Symbol("KEY"),
    result;
  context[key] = self;
  result = context[key](...params);
  Reflect.deleteProperty(context, key);
  return result;
};

var name = "珠峰培训";
function A(x, y) {
  var res = x + y;
  console.log(res, this.name);
}
function B(x, y) {
  var res = x - y;
  console.log(res, this.name);
}
B.call(A, 40, 30);
// 把call方法执行 this->B  context->A  params->[40,30]
// A[key]=B  A[key](40,30)  把B执行，让B中的this指向A，传递实参40/30  => 10 "A"

B.call.call.call(A, 20, 10);
// 把最后一个call方法执行  this->B.call.call(就是个call方法)  context->A  params->[20,10]
// “A[key]”=call方法  “A[key]”(20,10)
// -> 继续把“call方法执行”   this->A  context->20  params->[10]
//  “new Number(20)[key]” = A;   “new Number(20)[key]”(10)
// 让A执行，A中的this指向20，传递实参10  => NaN undefined

Function.prototype.call(A, 60, 50);
// 直接最后：把Function.prototype执行(匿名空函数)，让其中的this指向A，传递60/50  => 无输出

Function.prototype.call.call.call(A, 80, 70);
// “Function.prototype.call.call” -> “call方法 @CA”
// 最后一个CALL执行：让@CA执行，让里面的this是A，传递80/70
// 80.xxx = A;  80.xxx(70);
// 把A执行，A中的this是80，传递实参70 => NaN undefined

// 规律：
// 一个CALL执行 例如：B.call(A, 40, 30);
// => 让B执行，让B中的this是A，给B传递40/30
// 多个CALL执行 例如：B.call.call.call(A, 20, 10);
// => 让A执行，让A中的this是20，给A传递10
```

## 14. ES6 的继承和 ES5 的继承有什么区别？

标准回答：

结论上，ES6 `extends` 是 ES5 原型继承的语法糖，但表达更清晰。

区别：

- `class/extends` 语义更直观
- `super` 使用更规范
- 子类构造函数中必须先 `super()` 后用 `this`
- 静态方法继承表达更自然

底层本质依然是原型链继承。

## 15. 函数的多种角色综合题答案

题目输出：

```js
2;
4;
1;
1;
2;
3;
3;
```

解释：

### `Foo.getName()`

访问的是函数对象 `Foo` 身上的静态方法，所以输出 `2`。

### `getName()`

变量提升后，函数声明先可用，但执行阶段：

```js
var getName = function () {
  console.log(4);
};
```

覆盖了函数声明，因此输出 `4`。

### `Foo().getName()`

`Foo()` 作为普通函数执行：

- 内部把全局 `getName` 改成输出 `1` 的函数
- 返回 `this`
- 非严格模式下 `this` 是 `window`

所以等价于：

```js
window.getName();
```

输出 `1`。

### `getName()`

前一步已经把全局 `getName` 改写成输出 `1` 的函数，所以再次输出 `1`。

### `new Foo.getName()`

运算符优先级下，等价于：

```js
new Foo.getName();
```

执行的是静态方法，输出 `2`。

### `new Foo().getName()`

等价于：

```js
new Foo().getName();
```

`new Foo()` 得到实例，实例本身没有 `getName`，沿原型链找到：

```js
Foo.prototype.getName;
```

输出 `3`。

### `new new Foo().getName()`

先算：

```js
new Foo();
```

得到实例，再取：

```js
new Foo().getName;
```

即原型上的那个函数，最后对它再 `new` 一次，本质执行的还是输出 `3` 的函数，所以结果还是 `3`。
