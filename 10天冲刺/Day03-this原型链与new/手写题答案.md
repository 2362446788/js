# Day 3 手写题答案

## 题 1：实现 `myCall`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myCall = function (context, ...args) {
  context = context == null ? globalThis : Object(context);
  const key = Symbol("fn");
  context[key] = this;
  const result = context[key](...args);
  delete context[key];
  return result;
};
```

详细思路：

1. `call` 的本质是：把函数临时放到目标对象上执行
2. 先把 `context` 处理成对象，空值默认指向全局对象
3. 用 `Symbol` 生成一个不冲突的临时 key
4. 挂上函数、执行、删除临时属性

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

## 题 2：实现 `myApply`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myApply = function (context, args) {
  context = context == null ? globalThis : Object(context);
  const key = Symbol("fn");
  context[key] = this;
  const result = Array.isArray(args) ? context[key](...args) : context[key]();
  delete context[key];
  return result;
};
```

详细思路：

- 和 `call` 完全同源，只是参数传递方式不同
- `apply` 的第二个参数是数组或类数组

## 题 3：实现 `myBind`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myBind = function (context, ...presetArgs) {
  const fn = this;

  function boundFn(...laterArgs) {
    const finalContext = this instanceof boundFn ? this : context;
    return fn.apply(finalContext, presetArgs.concat(laterArgs));
  }

  boundFn.prototype = Object.create(fn.prototype);
  return boundFn;
};
```

说明：

- 面试高频追问是“`new` 调用 bind 后函数怎么办”
- 所以要处理 `this instanceof boundFn`

详细思路：

1. `bind` 不立即执行，而是返回新函数
2. 新函数执行时，把预置参数和后续参数拼起来
3. 正常调用时，`this` 应该绑定到 `context`
4. 但如果是 `new boundFn()`，此时应该忽略外部绑定，让 `this` 指向新实例
5. 为了让实例还能沿原型链找到原函数原型上的方法，要重建原型链

复杂度：

- 时间复杂度：单次调用 `O(n)`，`n` 与参数合并有关
- 空间复杂度：`O(1)`

## 题 4：实现 `mockNew`

来源：

- 高频前端手写题
- Day03 `new` 核心原理

```js
function mockNew(Constructor, ...args) {
  const instance = Object.create(Constructor.prototype);
  const result = Constructor.apply(instance, args);
  const isObject = result !== null && (typeof result === "object" || typeof result === "function");
  return isObject ? result : instance;
}
```

详细思路：

1. 先创建一个新对象，并让它的原型指向构造函数原型
2. 用新对象作为 `this` 执行构造函数
3. 判断构造函数返回值：
  - 如果返回对象或函数，就用这个返回值
  - 否则返回新对象

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

## 题 5：实现 `myInstanceof`

来源：

- 高频前端手写题
- Day03 原型链专题

```js
function myInstanceof(obj, Constructor) {
  if (obj == null || (typeof obj !== "object" && typeof obj !== "function")) {
    return false;
  }

  let proto = Object.getPrototypeOf(obj);
  const targetProto = Constructor.prototype;

  while (proto) {
    if (proto === targetProto) return true;
    proto = Object.getPrototypeOf(proto);
  }

  return false;
}
```

详细思路：

1. `instanceof` 的本质是“看构造函数原型是否出现在对象原型链上”
2. 所以从 `obj` 的原型开始，一路向上找
3. 找到就返回 `true`
4. 找到 `null` 还没有，就返回 `false`

复杂度：

- 时间复杂度：`O(h)`，`h` 是原型链高度
- 空间复杂度：`O(1)`
