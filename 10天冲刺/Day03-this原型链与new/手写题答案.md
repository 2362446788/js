# Day 3 手写题答案

## 题 1：实现 `myCall`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myCall = function (context, ...args) {
  context = context == null ? globalThis : Object(context);
  const key = Symbol("fn");
  context[key] = this;
  const result = context[key](...args);
  delete context[key];
  return result;
};
```

详细思路：

1. `call` 的本质是：把函数临时放到目标对象上执行
2. 先把 `context` 处理成对象，空值默认指向全局对象
3. 用 `Symbol` 生成一个不冲突的临时 key
4. 挂上函数、执行、删除临时属性

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

## 题 2：实现 `myApply`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myApply = function (context, args) {
  context = context == null ? globalThis : Object(context);
  const key = Symbol("fn");
  context[key] = this;
  const result = Array.isArray(args) ? context[key](...args) : context[key]();
  delete context[key];
  return result;
};
```

详细思路：

- 和 `call` 完全同源，只是参数传递方式不同
- `apply` 的第二个参数是数组或类数组

## 题 3：实现 `myBind`

来源：

- 2026 Front End Interview Handbook `call` / `apply` / `bind`
- 高频前端手写题

```js
Function.prototype.myBind = function (context, ...args) {
  // 1. 边界处理：确保调用者是一个函数
  if (typeof this !== "function") {
    throw new TypeError("myBind must be called on a function");
  }

  // 保存原函数的引用（this 指向调用 myBind 的那个函数）
  const self = this;

  // 2. 创建一个中间函数（作为返回的新函数）
  const boundFunc = function (...innerArgs) {
    // 3. 判断是否通过 new 调用
    // 如果 this 是 boundFunc 的实例，说明是通过 new 调用的构造函数
    const isNewCall = this instanceof boundFunc;

    // 如果是 new 调用，this 指向新实例；否则指向 bind 传入的 context
    const finalThis = isNewCall ? this : context || globalThis;

    // 4. 合并参数：bind 时预设的参数 + 调用时传入的参数
    const finalArgs = [...args, ...innerArgs];

    // 5. 执行原函数并返回结果
    return self.apply(finalThis, finalArgs);
  };

  // 6. 维护原型链：让新函数的 prototype 指向原函数的 prototype
  // 这样通过 new boundFunc() 创建的实例，能正确继承原函数原型上的属性和方法
  if (self.prototype) {
    boundFunc.prototype = Object.create(self.prototype);
  }

  // 7. 返回绑定了上下文的新函数
  return boundFunc;
};
```

说明：

- 面试高频追问是“`new` 调用 bind 后函数怎么办”
- 所以要处理 `this instanceof boundFn`

详细思路：

1. `bind` 不立即执行，而是返回新函数
2. 新函数执行时，把预置参数和后续参数拼起来
3. 正常调用时，`this` 应该绑定到 `context`
4. 但如果是 `new boundFn()`，此时应该忽略外部绑定，让 `this` 指向新实例
5. 为了让实例还能沿原型链找到原函数原型上的方法，要重建原型链

复杂度：

- 时间复杂度：单次调用 `O(n)`，`n` 与参数合并有关
- 空间复杂度：`O(1)`

## 题 4：实现 `mockNew`

来源：

- 高频前端手写题
- Day03 `new` 核心原理

```js
function mockNew(Constructor, ...args) {
  const instance = Object.create(Constructor.prototype);
  const result = Constructor.apply(instance, args);
  const isObject =
    result !== null &&
    (typeof result === "object" || typeof result === "function");
  return isObject ? result : instance;
}
```

详细思路：

1. 先创建一个新对象，并让它的原型指向构造函数原型
2. 用新对象作为 `this` 执行构造函数
3. 判断构造函数返回值：

- 如果返回对象或函数，就用这个返回值
- 否则返回新对象

复杂度：

- 时间复杂度：`O(1)`
- 空间复杂度：`O(1)`

## 题 5：实现 `myInstanceof`

来源：

- 高频前端手写题
- Day03 原型链专题

```js
const myInstanceof = function myInstanceof(obj, ctor) {
  // 检测值类型的校验：校验ctor的格式
  if (ctor === null || !/^(object|function)$/.test(typeof ctor))
    throw new TypeError("Right-hand side of instanceof is not an object");
  if (typeof ctor !== "function")
    throw new TypeError("Right-hand side of instanceof is not callable");
  if (!ctor.prototype)
    throw new TypeError(
      "Function has non-object prototype undefined in instanceof check",
    );
  // 检测值类型的校验：不支持原始值类型
  if (obj === null || !/^(object|function)$/.test(typeof obj)) return false;
  // 首先检测ctor是否拥有Symbol.hasInstance方法
  if (typeof Symbol !== "undefined") {
    let hasInstance = ctor[Symbol.hasInstance];
    if (typeof hasInstance === "function") {
      return hasInstance.call(ctor, obj); //=> ctor[Symbol.hasInstance](obj);
    }
  }
  // 按照原型链进行查找，看是否会出现ctor.prototype
  let proto = Object.getPrototypeOf(obj);
  while (proto) {
    if (proto === ctor.prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
  return false;
};
```

详细思路：

1. `instanceof` 的本质是“看构造函数原型是否出现在对象原型链上”
