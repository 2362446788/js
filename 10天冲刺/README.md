## 这是新的唯一学习入口

从现在开始，统一从这个目录学习，不再从散落的课件目录直接进入。

## 为什么要重排

原来的资料更像“上课资料归档”，不适合 10 天冲刺。

现在重排后的原则是：

1. 先学最底层、最影响后续理解的内容
2. 再学高频、高难、最容易挂掉的内容
3. 再补工程化、浏览器、Vue 联调
4. 最后两天只做模拟和查漏

## 使用方式

每天固定按这个顺序学习：

1. 先看当天的 `知识点.md`
2. 再看对应原始课件和目录
3. 再做 `手写题.js`
4. 学完后刷 `面试题与回顾.md`
5. 最后对照 `面试题答案.md` 和 `手写题答案.md`

## 学习节奏建议

每天建议固定 8 小时左右：

- 上午 3 小时：看 `知识点.md` + 原始课件
- 下午 3 小时：刷 `面试题与回顾.md` + 手写
- 晚上 2 小时：对照 `面试题答案.md` 复述和纠错

## 难度排序说明

这 10 天不是按课件日期学，而是按面试最合理的难度递进来排：

1. 先打牢最基础的类型、作用域、原型
2. 再上高难核心的 Promise / Event Loop
3. 再补工程化和浏览器专题
4. 最后两天只做模拟与复盘，不再吃新知识

## 每天最低完成标准

- `知识点.md` 过完
- 原始资料至少看完主文件
- `手写题.js` 至少独立写 2 到 4 题
- `面试题与回顾.md` 过一遍
- `面试题答案.md` 对答案并补口述
- `手写题答案.md` 对照修正实现

## 最关键的策略

不是把资料都看完，而是把高频题练到：

- 你能讲
- 你能写
- 你能结合项目说

做到这三点，比“多看三份资料”更重要。
